# 기여 및 운영 원칙

이 저장소는 iDAI.field를 한국 매장문화재 조사 현장에 맞게 조정하는 독립 포크입니다. 원본 프로젝트의 출처와 라이선스는 분명히 남기되, 개발과 운영 책임은 이 저장소 안에서 해결합니다.

## 독립 포크 원칙

- 원작자가 이 포크의 변경을 Pull Request로 받을 수 없다고 밝힌 상황을 존중합니다.
- 이 저장소의 기능 변경, 버그 수정, 한국형 조사 워크플로, 태블릿 배포 문제는 이 저장소의 이슈, 브랜치, 커밋, 릴리스로 처리합니다.
- 원본 iDAI.field 저장소에는 사전 합의 없는 Pull Request, 리뷰 요청, 멘션, 사용자 지원 요청을 보내지 않습니다.
- 원본 프로젝트에는 출처, 라이선스, 공개 문서 링크, 필요한 upstream 참고 기록만 남깁니다.

## 원본 프로젝트 예우

원본 프로젝트는 이 포크의 유지보수 창구가 아닙니다. 원작자와 원본 유지관리자의 시간을 쓰게 만드는 방식으로 문제를 넘기지 않습니다. upstream 변경을 참고해야 할 때도 먼저 이 저장소 안에서 검토하고, 필요한 경우에만 공개 이력으로 출처를 남깁니다.

## 작업 흐름

1. 한국 현장용 변경은 이 저장소의 작업 브랜치에서 진행합니다.
2. 태블릿과 데스크톱에 함께 영향을 주는 변경은 가능한 한 같은 커밋 또는 같은 작업 묶음에서 검증합니다.
3. 공개 배포 전에는 관련 테스트와 한국 현장 워크플로 검사를 통과시킵니다.
4. 태블릿 검증용 스크린샷, 임시 XML, 개인 API 키, 현장 원자료는 커밋하지 않습니다.
5. 원본 iDAI.field와의 관계를 설명할 때는 출처와 라이선스를 밝히되, 이 포크를 원본 프로젝트의 공식 배포판처럼 표현하지 않습니다.
