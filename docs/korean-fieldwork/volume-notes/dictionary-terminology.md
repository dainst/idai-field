# 고고학사전 용어 표준화 노트

대상 자료군: `C:\Users\nuri9\Downloads\고고학사전`

이 노트는 사전류 PDF를 iDAI.field 한국형 야장의 화면명, 값 목록, 도움말, 보고서 출력 용어로 옮기기 위한 작업 공간이다. 사전 원문을 옮겨 적는 것이 아니라, 용어를 어떻게 표준화하고 동의어를 어떻게 다룰지 정리한다.

## 1차 확인

추출 텍스트는 모두 `C:\Users\nuri9\AppData\Local\Temp\codex-idai-field-new-pdf-text`에 있다. 2001년 `한국고고학사전` 1·2권은 직접 추출이 되지만 공백과 일부 글자가 많이 무너져 있어 넓은 용례 확인용으로 먼저 쓰고, 실제 UI 표제어는 전문사전과 원 PDF 화면 대조 뒤 확정한다.

전문사전류는 분야별 표제어 체계가 비교적 분명하다. 특히 생산유적편은 토기, 기와, 자기, 도기, 제철, 목탄, 공방 및 기타처럼 분야를 나누고, 한뎃가마와 노천요처럼 이칭과 우선 표제어를 연결하는 방식이 확인된다. 청동기시대편 증보판은 유적·유구·유물 명칭, 단위 표기, 개념 표제어 처리 원칙을 안내하므로 값 목록과 측정 단위 기본값을 정리하는 데 쓸 수 있다.

## 용어 표준화 원칙

1. 전문사전의 표제어를 곧바로 필드명으로 쓰지 않는다. 먼저 `우선표제어`, `동의어`, `이칭`, `기관관용어`, `보고서출력어`를 분리한다.
2. 같은 말이 분야마다 다른 범위를 가질 수 있으므로 용어에는 시대, 유적 유형, 유물 재질, 조사 단계, 출처 사전을 함께 붙인다.
3. 사전의 이칭 관계는 검색어와 자동완성에는 적극 사용하되, 저장값은 우선표제어와 출처 ID를 남긴다.
4. 현장기관의 관용어가 전문사전 표제어와 다르면 사용자를 막지 않는다. 대신 보고서 출력 시 우선표제어로 바꾸거나 병기할 수 있게 한다.
5. 2001년 일반 사전처럼 추출 품질이 흔들리는 자료는 `원문대조필요` 상태로 둔다. UI에 바로 노출할 표준값은 최신 전문사전, 해당 분야 전문사전, 실제 보고서 관용례를 대조해 확정한다.
6. 사전류는 기록 항목의 어휘를 정리하는 근거이지, 현장 필수 입력값을 늘리는 근거가 아니다. 필수값 여부는 조사방법론 자료와 보고서 역산 요구에서 판단한다.

## 생산유적편·청동기시대편 1차 대조

생산유적편은 토기, 기와, 자기, 도기, 제철, 목탄, 공방 및 기타처럼 분야를 먼저 나누고, 사전 내부에서 이칭과 관련 표제어를 서로 참조한다. 이 구조는 `가마/요/요지`, `한뎃가마/노천요`, `수비장/토기공방`, `건조장/토기공방`처럼 검색어와 저장 표제어가 달라질 수 있음을 보여준다.

청동기시대편 증보판은 유적, 유구, 유물, 문화유형, 개념 표제어를 함께 다루며, 국내에서 쓰이는 표준 명칭이 있는 경우 외국 자료명도 한국어 관용 표기를 따른다. 유구는 m, 유물은 cm처럼 기본 단위가 다르게 제시되므로, 화면별 단위 기본값을 용어 권위 정보와 분리하지 말아야 한다.

필드 후보:

- `용어관계종류`: 동의어, 이칭, 상위어, 하위어, 기능유사, 구조유사, 시대한정, 분야한정, 원문대조필요, 판단 보류
- `생산유적분야표지`: 토기, 기와, 자기, 도기, 제철, 목탄, 공방기타, 개념, 판단 보류
- `용어측정단위적용`: 유구_m, 유물_cm, 면적_m2, 해발_m, 좌표별도, 예외기록, 판단 보류

자동완성은 이칭을 넓게 받아야 하지만 저장값은 우선표제어, 분야, 시대, 출처, 검증상태로 정규화한다. `번조실`과 `소성실`처럼 기능과 구조가 겹치는 말은 단순 동의어로 합치지 않고, 현장 화면에서는 선택어와 설명어를 분리한다.

## iDAI.field 반영 방향

1. `TermAuthority`를 새로 두고 우선표제어, 한자, 영문, 분야, 시대, 출처, 검증상태를 저장한다.
2. `TermAlias`는 동의어와 이칭을 저장하되, 저장 데이터의 기본값은 `TermAuthority`를 향하게 한다.
3. 기존 값 목록에는 한국어 표시값만 넣지 말고 `sourceDictionary`, `sourcePage`, `verificationState`, `domain`을 붙인다.
4. 도움말은 사전 정의를 길게 번역하지 않고, 현장 입력에 필요한 구분 기준과 혼동 위험만 요약한다.
5. 용어 표준화는 한 번에 전체 사전을 값 목록으로 옮기는 방식이 아니라, 실제 템플릿을 만들 때 필요한 분야부터 부분 적용한다.
