# 한국형 공통 기록 모델 v0

이 문서는 『한국 매장문화재 조사연구방법론』 제1권 총론에서 드러나는 문제의식과 현재 iDAI.field 구조를 연결해 만든 공통 기록 모델의 첫 초안이다. 이후 권별 분석이 진행될수록 범주와 필드는 추가·수정한다.

## 설계 원칙

한국형 야장은 하나의 거대한 입력 양식이 아니라 “공통 핵심 + 유적 성격별 확장”이어야 한다. 제1권 목차가 총론, 제도, 건물지, 제철유적, 고분, 자연과학 분석, 보존·정비, 현장수습, 지형·입지, 정보화를 함께 다루는 것은 발굴조사가 단일 행위가 아니라 행정, 현장 관찰, 자료 수습, 분석, 보존, 보고, 정보화가 이어지는 흐름임을 보여준다.

현장 기록은 관찰 사실과 해석을 분리하고, 해석 근거와 불확실성을 함께 남겨야 한다. 특히 제1권의 발굴조사 개선 논의와 제7권의 성곽 축조기법 논의는 명칭을 먼저 붙이는 기록보다 “무엇을 근거로 그렇게 판단했는가”를 남기는 기록이 중요하다는 점을 보여준다.

따라서 iDAI.field에서 한국형 기본 모델은 다음 네 가지를 동시에 다뤄야 한다.

- 현장 관찰: 유구, 층위, 유물, 시료, 지형, 관계
- 조사 행위: 조사일, 작업자, 작업공정, 수습, 보존, 검토
- 증거 매체: 사진, 도면, GIS, 3D, 분석 결과
- 보고 산출물: 유구 설명, 유물 목록, 사진대장, 도면 목록, 시료 분석표, 보존 기록

## 공통 범주 후보

| 범주 | 목적 | iDAI.field 매핑 후보 |
| --- | --- | --- |
| 조사사업 | 허가, 계약, 조사 목적, 기간, 기관 정보를 묶는다 | `Project` 확장 또는 별도 `Investigation` |
| 유적 | 조사 대상의 상위 공간 단위 | `Place` 또는 `Feature` 상위 범주 |
| 조사구역 | 지점, 구역, 그리드, 트렌치, 피트 등을 관리한다 | `Operation`/`Place` 확장 |
| 층위 | 퇴적 단위와 층서 관계를 기록한다 | `StratigraphicUnit` 후보 |
| 유구 | 건물지, 고분, 생산유구, 성곽 등 관찰 단위 | `Feature` 하위범주 |
| 유물 | 출토 맥락과 연결되는 물질 자료 | `Find` 하위범주 |
| 시료 | 자연과학 분석과 연결되는 채취 단위 | `Sample` 하위범주 신설 검토 |
| 사진 | 촬영 대상, 방향, 촬영자, 파일을 연결한다 | `Image` 확장 |
| 도면 | 실측도, 평면도, 단면도, 유구도 등을 연결한다 | `Drawing` 또는 `Document` 범주 신설 |
| 작업기록 | 조사일지와 당일 작업 맥락 | `Workflow`/`Operation` 확장 |
| 수습/보존조치 | 현장수습, 응급처치, 반출, 보존 필요성을 기록한다 | `ConservationAction` 범주 신설 |
| 분석결과 | 자연과학 분석 의뢰와 결과를 추적한다 | `Analysis` 범주 신설 |
| 조사기관 | 기관 성격, 등록범위, 전문인력, 역할분담, 공익성·투명성 기록 | `InstitutionRecord` 후보 |
| 행정결정 | 허가, 협의, 단계 전환, 지도위원회, 행정절차 처리 | `AdministrativeDecision` 후보 |
| 보고서평가 | 보고서 발간 지침, 심사평가, 보완, 공개 상태 | `ReportEvaluation` 후보 |
| 정보화자산 | 전자보고서, 작업 파일, 사진·도면 DB, 재래식 매체 목록 | `InformationAsset` 후보 |

## 보고서 역산 체크리스트

야장 입력 화면은 다음 질문에 답할 수 있어야 한다.

- 이 유구 설명의 근거가 되는 원 현장기록은 무엇인가?
- 이 사진은 어떤 유구/층위/유물/작업 상태를 보여주는가?
- 이 유물은 어떤 맥락에서 수습되었고 어떤 대장 항목으로 출력되는가?
- 이 시료는 어떤 지점에서 어떤 목적으로 채취되었고 어떤 분석 결과와 연결되는가?
- 이 도면의 대상, 축척, 기준점, 작성자, 검토 상태는 무엇인가?
- 이 해석은 관찰 사실과 어떻게 분리되어 있는가?
- 보고서 작성 중 수정된 해석과 현장 원기록의 차이가 남아 있는가?

## iDAI.field 적용 우선순위

1. `Sample`, `Drawing`, `Analysis`, `ConservationAction`, `DailyLog` 범주의 필요성을 검토한다.
2. 기존 `Feature`, `Find`, `Image`, `Operation` 범주에 한국형 필드 확장을 붙인다.
3. 값 목록을 한국어 현장 용어 중심으로 재구성한다.
4. 샘플 프로젝트에서 “유구 1건이 보고서 표와 사진대장으로 이어지는 흐름”을 검증한다.
5. 현장 입력 시간이 길어지지 않도록 필수/선택 필드를 분리한다.
