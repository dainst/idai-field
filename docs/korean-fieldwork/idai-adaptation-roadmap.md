# iDAI.field 접목 로드맵

이 문서는 한국형 야장 원칙을 iDAI.field에 어떻게 접목할지 정리하는 실행 로드맵이다.

## 1단계: 분석 저장소 만들기

현재 단계에서는 책 원문을 저장소에 넣지 않고, 권별 분석 노트를 만든다.

예정 구조:

```text
docs/korean-fieldwork/
  source-inventory.md
  field-recording-principles.md
  field-observation-matrix.md
  field-recording-workflows.md
  idai-adaptation-roadmap.md
  volume-notes/
    volume-01-general.md
    volume-02-prehistoric-sites.md
    volume-03-building-sites.md
    volume-04-production-sites.md
    volume-05-tombs.md
    volume-06-alluvial-sites.md
    volume-07-fortresses.md
    volume-08-neolithic-sites.md
    volume-09-research-methods.md
    volume-2020-iron-production.md
```

각 권별 노트는 다음 형식을 따른다.

- 출처와 서지
- 목차
- 현장에서 관찰해야 할 대상
- 기록 단위
- 필수 필드 후보
- 값 목록 후보
- 사진/도면/GIS/시료 요구사항
- 보고서 산출물과의 연결
- iDAI.field 구성 변경 제안

## 2단계: 한국형 공통 핵심 모델 정의

iDAI.field의 기존 `core`는 유지하되, 한국 현장에 맞는 `Library` 확장을 설계한다.

공통 모델 후보:

- 조사사업
- 유적/지점
- 조사구역
- 그리드/트렌치
- 층위
- 유구
- 유물
- 시료
- 사진
- 도면
- 보존/응급처치 기록
- 일일 작업기록

## 5단계: 입력 흐름 설계

한국형 야장은 현장 작업 속도를 방해하면 실패한다. 따라서 화면 설계는 다음 순서를 우선한다.

1. 오늘의 작업기록 생성
2. 조사구역/트렌치 선택
3. 층위 또는 유구 생성
4. 관찰 사실 입력
5. 사진/도면/GIS 연결
6. 유물/시료 수습
7. 불확실성 및 확인 필요 사항 표시
8. 검토자 확인

실제 구현 착수 전에는 [한국형 야장 구현 요구사항](field-notebook-requirements.md)의 첫 구현 단위부터 확인한다. 현재 `Config-KoreanFieldwork`에는 기록 품질검수, 독립 `FieldRecordQualityReview`, `DailyLog`, `TermAuthority` 카드, 현장시점 누락점검, 디지털 원자료 보존, 보고서 역산 교차검토, 보고서 평가 환류에 더해 조사사업 행정 타임라인, 허가조건 이행, 보존조치 이행, 조사자료 인계, 지표조사 현장관찰·편향방지·후속조치, GIS·항공사진 예측근거, 예측 현장검증, 용어 관계·검색매핑·검증상태, 사전 분야, 용어 적용범위, 출처 우선순위, 시료 채취·보관, 일일 작업기록, 일지 증거 역할, 일지 검토, 형식 논증, 편년 논증, 공반 관계, 매체 증거 역할, 매체 품질검수 값 목록까지 들어갔다. 현장시점 누락점검과 최초 노출 기록은 필수값으로 지정해 기존 경고 UI의 `missingMandatoryFields` 흐름에 먼저 연결했고, `Feature`와 `Find`에는 형식·편년·공반 논증을, `Survey`에는 지표조사 관찰·편향방지·후속조치와 GIS 예측 현장대조를, `Photo`와 `Drawing`에는 매체 증거 역할·품질검수·원자료 보존·보고서 교차검토를 배치했다. 이후 보강으로 `Survey`에 표본조사 적합성, 시굴조사 목적, 시굴 트렌치 설계, 정밀발굴 범위·난이도 근거를 추가했고, `Sample`에는 식물고고학 시료 설계, 식물유체 표본추출, 플로테이션 처리기록, 식물유체 동정기록, 식물고고학 해석검토, 식물유체 미검출 평가를 추가했다. 다음 우선순위는 `TermAuthority` 검색 인덱스/자동완성, 샘플 프로젝트 검증, 기록시점에 따른 조건부 경고 정교화다.

## 6단계: 샘플 프로젝트 검증

기본 템플릿이 만들어지면 가상의 한국 발굴 현장을 샘플 프로젝트로 구성한다.

검증 질문:

- 하루 작업기록에서 보고서 초안까지 추적되는가?
- 사진대장과 도면 목록을 자동으로 만들 수 있는가?
- 유물/시료가 출토 맥락과 분리되지 않는가?
- 층위 관계와 유구 관계를 명확히 입력할 수 있는가?
- 현장 조사자가 긴 설명 없이 쓸 수 있는가?

## 7단계: 실제 구성 파일 반영

분석 문서가 충분히 쌓인 뒤 다음 파일군을 수정한다.

현재 `Sample` 화면 보강은 식물유체·탄화곡물 묶음에 이어 동물유체·화석환경 묶음까지 들어갔다. 동물유체 구현은 `faunalRecoverySampling`, `faunalPreservationHandling`, `zooarchaeologicalIdentification`, `boneSurfaceModification`, `zooarchaeologicalQuantification`으로 수습·보존·동정·표면변형·계량지표를 분리하고, 큰 뼈 선별 수습과 소형자료 누락, 블록샘플 치수, 체눈, 보존 환경, NISP·MNE·MNI·MAU·%MAU 한계를 함께 기록하도록 한다.

`Survey` 화면의 고지형·시굴 전환 보강에는 저습지 조사방법도 추가했다. `wetlandAnalysisSource`, `wetlandLandformInterpretation`, `wetlandSurveyTargeting`은 항공사진·고지도·보링자료 준비, 지역환경에서 극미지형까지의 판독 단계, 고하천·자연제방·단구면·배후습지 표시와 시굴 지점 선정을 기록한다. `FeatureSegment`의 `wetlandMicrotopographyRecord`는 실제 발굴에서 확인되는 1회성 퇴적, 수전 매몰토층, 미화석 분석, 목제품·종자·동물유체 맥락을 층 단위로 다시 붙인다.

조사기초과정의 층 구분·유구 내부토 보강은 `FeatureSegment`의 `stratigraphicDivisionBasis`, `layerNamingSystem`, `featureFillInterpretation`, `soilTextureFieldAssessment`로 추가했다. 색·층명 중심 기록을 피하고, 입도·퇴적구조·삭평·토양화·구지표·문화층, 층 번호체계와 변경 이력, 가공면·기능면·폐기 후 퇴적층과 유물 귀속 주의, 손가락 비빔·구슬/리본·표본시료 대조 같은 토성 판정법을 같은 층 세부단위에서 관리한다.

생활유적 조사절차 보강은 `Feature`의 `pitFeatureFunctionAssessment`, `settlementFeatureInvestigationProcedure`, `settlementFeatureTrenchStrategy`로 추가했다. 수혈유구의 주거·창고·공방·공공시설·함정·화장실·폐기장·태토 채취장 후보를 열어두고, 조사 절차는 사전조사부터 바닥조사·절개까지 단계화하며, 트렌치 전략은 최소 훼손, 바닥면·벽선 보존, 일자·십자·교차식 선택, 축 방향, 등고선 관계, 단면 연속성과 도면 역전 위험을 함께 검토하게 한다.

`Drawing` 화면에는 도면작성법 보강으로 `mapSourceMaterial`, `historicalMapLandscapeInterpretation`, `spatialDrawingProductionWorkflow`, `distributionMapRequirement`를 추가했다. 완성 도면 파일만 저장하지 않고 고지도·지형도·항공사진·DEM 같은 원천자료, 지도에서 읽은 경관 단서, 좌표계 변환과 객체 선택, 주변유적분포도 반경·범례·산출물 요건을 같은 도면 카드에서 검수한다.

GPS·GIS 현장 시스템 보강은 `Operation`과 `Photo`에 나누어 넣었다. `gpsSurveyQualityRecord`, `gpsNmeaRecord`, `fieldDatabaseOperationRisk`는 작업 단위의 측량 품질과 현장 DB 운영 위험을 관리하고, `gpsPhotoLinkRecord`는 사진 파일명과 GPS 궤적·현재위치·위치 자동분류 연결을 사진 카드에서 검수한다.

지표조사 실무 보강은 `Survey`와 `Find`에 나누어 넣었다. `surfaceSurveyPreparationCheck`, `surfaceSurveyFieldSequence`, `surfaceSurveyMapRequirement`는 조사단·전문가·문헌·지도·장비 준비, 경계 GPS와 전경 사진, 지형별 지구 구분, 기존 문화재·유물산포·노출 유구·절개면 토층 확인, 다각형 조사범위와 GPS 기준점이 있는 지도요건을 관리한다. `surfaceFindHandlingRecord`는 지표 수습 유물의 개별 포장, 유물카드, 지도·GPS·지번, 근경·원경 사진, 부착토 보존과 세척주의를 `Find`에서 추적한다.

지표조사 범위와 조사단 구성 보강도 `Survey`에 추가했다. `surfaceSurveyHeritageCategory`, `surfaceSurveyScopeDefinition`, `surfaceSurveyLowerChronologyReview`, `surfaceSurveyTeamExpertise`, `surfaceSurveyTimingReview`는 고고·건축사·역사·민속·자연유산·수중·근대문화유산 같은 조사대상 범주, 사업구역과 주변 영향권, 근현대 하한연대, 분야별 전문가 필요성, 사업 단계별 조사 시점과 보존 검토 시간 확보 여부를 관리한다.

지표 증거부재와 비유적지 자원조사는 `surfaceEvidenceAbsenceAssessment`, `nonSiteResourceSurvey`로 `Survey`에 추가했다. 표면 증거 없음은 유적 없음으로 자동 확정하지 않고 지형·입지·퇴적물·탐문·경험 판단·조사한계·추가조사 필요로 분해하며, 주변 석재·점토·성곽석재·고인돌석재·생산 원료 후보는 비유적지 조사값으로 남긴다.

전자도면·3D 스캔 보강은 `Drawing`에 `electronicDrawingSourceWorkflow`, `artifactElectronicDrawingProcedure`로 넣었다. 전자는 스캐너 선정, 점군 획득·병합, 폴리곤 변환·최적화, CAD·2D·3D·복원 산출물과 파일 보존을 남기고, 후자는 유물 형태 검토, 특징 view, 단면 위치, 벡터화, 기준면, 좌표계 이동, 합치 기준처럼 유물 전자도면의 연구자 판단을 남긴다.

- `core/config/Library/Valuelists/Language.projects.ko.json`

이 단계부터는 단순 번역이 아니라 데이터 모델 변경이므로, 샘플 데이터와 테스트가 함께 필요하다. 1차 반영에서는 한국형 공통 값 목록을 프로젝트 전용 목록으로 추가했고, `Config-KoreanFieldwork.json`에서 Project, Operation, Survey, FeatureGroup, Feature, FeatureSegment, Find, Sample, Photo, Drawing 화면에 연결했다. 이번 보강에서는 현장기록 품질, 현장시점 누락점검, 개인 야장 공적기록화, 디지털 원자료 보존, 일일 작업기록, 일지 검토, 용어 검색·매핑 값을 실제 구성에 추가했다. 후속 보강으로 성곽 성문·방어 부속시설, 수문·수구, 왜성 해자 값을 `Feature` 화면에 올렸고, 보고서 평가 환류를 `Project`와 `Operation` 화면에 추가했다. `Survey` 화면에는 지표조사 현장관찰·편향방지·후속조치와 GIS·항공사진 예측근거·현장검증을 연결했다. `Photo`와 `Drawing` 화면에는 매체 증거 역할, 품질검수, 디지털 원자료 보존, 보고서 교차검토를 연결했다. 이번 단계에서는 `Operation` 하위 `FieldRecordQualityReview` 카드를 추가해 검수 대상 기록, 검수 단계, 수정·보완 근거를 원기록과 분리하고, `DailyLog` 카드를 추가해 당일 사실기록과 일지의 증거 역할을 별도 기록하게 했다. 이어 `FeatureGroup` 하위 `TermAuthority` 카드를 추가해 사전 분야, 적용범위, 출처 우선순위, 관계·검색·검증상태를 값목록 승격 전 단계에서 관리하게 했다. 프로젝트 식별자 `korean-fieldwork`는 `KoreanFieldwork` 구성을 불러온다.
