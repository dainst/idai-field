# iDAI.field 접목 로드맵

이 문서는 한국형 야장 원칙을 iDAI.field에 어떻게 접목할지 정리하는 실행 로드맵이다.

## 1단계: 분석 저장소 만들기

현재 단계에서는 책 원문을 저장소에 넣지 않고, 권별 분석 노트를 만든다.

예정 구조:

```text
docs/korean-fieldwork/
  source-inventory.md
  field-recording-principles.md
  field-observation-matrix.md
  field-recording-workflows.md
  idai-adaptation-roadmap.md
  volume-notes/
    volume-01-general.md
    volume-02-prehistoric-sites.md
    volume-03-building-sites.md
    volume-04-production-sites.md
    volume-05-tombs.md
    volume-06-alluvial-sites.md
    volume-07-fortresses.md
    volume-08-neolithic-sites.md
    volume-09-research-methods.md
    volume-2020-iron-production.md
```

각 권별 노트는 다음 형식을 따른다.

- 출처와 서지
- 목차
- 현장에서 관찰해야 할 대상
- 기록 단위
- 필수 필드 후보
- 값 목록 후보
- 사진/도면/GIS/시료 요구사항
- 보고서 산출물과의 연결
- iDAI.field 구성 변경 제안

## 2단계: 한국형 공통 핵심 모델 정의

iDAI.field의 기존 `core`는 유지하되, 한국 현장에 맞는 `Library` 확장을 설계한다.

공통 모델 후보:

- 조사사업
- 유적/지점
- 조사구역
- 그리드/트렌치
- 층위
- 유구
- 유물
- 시료
- 사진
- 도면
- 보존/응급처치 기록
- 일일 작업기록

## 3단계: 범주별 확장 템플릿 설계

권별 분석 결과를 바탕으로 템플릿을 분리한다.

| 템플릿 | 근거 권 | iDAI.field 적용 방식 |
| --- | --- | --- |
| 건물지 기록 템플릿 | 1, 3 | 유구 하위범주, 건축 요소, 유물 원위치, 건축부재 속성, 초석 흔적, 칸·영조척, 목조건축 구조, 복원 근거, 동시성·기능 판단 필드 추가 |
| 생산유적 기록 템플릿 | 1, 4 | 노/가마/작업장/부산물/시료 필드 추가 |
| 고분 기록 템플릿 | 1, 5 | 매장주체부, 봉토, 부장품 위치군, 제사자료, 인골 수습 관계 모델 추가 |
| 고지형/충적지 템플릿 | 6 | 지형 단위, 퇴적 단위, 미지형 입지모델, 시굴 트렌치, 주상도, 장비 제토, 환경 시료, 유구 관계망 필드 추가 |
| 선사·석기 산포 템플릿 | 2 | 석기 판정, 3차원 수습, 접합군, 도면 검토, 제작기법 필드 추가 |
| 성곽 기록 템플릿 | 7 | 성벽 구간, 축조기법, 문지/해자 관계 추가 |
| 신석기 유적 템플릿 | 2, 8 | 패총/주거지/생계자료/환경자료 필드 추가 |
| 연구·분석 템플릿 | 9 | 연구 질문, 분석 의뢰, 공개자료 등급 추가 |
| 제철유적 특화 템플릿 | 4, 2020 별책 | 제철 공정, 부산물, 금속학 시료, 복원실험 연결 추가 |

우선 구현 순서는 `공통 핵심 모델 -> 건물지/고분/생산유적 -> 충적지/성곽/신석기 -> 연구·분석/제철 특화`로 둔다. 2·3·5·6권은 스캔본이라 후속 OCR 후 세부 필드를 보강한다.

## 5단계: 입력 흐름 설계

한국형 야장은 현장 작업 속도를 방해하면 실패한다. 따라서 화면 설계는 다음 순서를 우선한다.

1. 오늘의 작업기록 생성
2. 조사구역/트렌치 선택
3. 층위 또는 유구 생성
4. 관찰 사실 입력
5. 사진/도면/GIS 연결
6. 유물/시료 수습
7. 불확실성 및 확인 필요 사항 표시
8. 검토자 확인

## 6단계: 샘플 프로젝트 검증

기본 템플릿이 만들어지면 가상의 한국 발굴 현장을 샘플 프로젝트로 구성한다.

검증 질문:

- 하루 작업기록에서 보고서 초안까지 추적되는가?
- 사진대장과 도면 목록을 자동으로 만들 수 있는가?
- 유물/시료가 출토 맥락과 분리되지 않는가?
- 층위 관계와 유구 관계를 명확히 입력할 수 있는가?
- 현장 조사자가 긴 설명 없이 쓸 수 있는가?

## 7단계: 실제 구성 파일 반영

분석 문서가 충분히 쌓인 뒤 다음 파일군을 수정한다.

- `core/config/Library/Language.ko.json`
- `core/config/Library/Templates/Language.ko.json`
- `core/config/Library/Valuelists/Language.default.ko.json`
- 필요 시 한국형 별도 configuration 세트

이 단계부터는 단순 번역이 아니라 데이터 모델 변경이므로, 샘플 데이터와 테스트가 함께 필요하다.
